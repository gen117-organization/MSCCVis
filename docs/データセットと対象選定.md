# データセットと対象選定

## 入力データ
- `dataset/` 配下のデータセットを元に分析対象を選定する。
- 実際の選定結果は `dest/` 配下に出力される。

## 対象プロジェクトの条件
- 同一プログラミング言語のコードベースが、サービス間にまたがって存在すること。
- マイクロサービス検出の結果を利用し、言語ごとのサービス数が2以上のものを対象とする。

## 分析対象コミット
- 対象コミットの選び方は設定値により切り替える。
- 例: 一定間隔で遡る / タグから選ぶ / merge commit から選ぶ。
- 実際の選定結果は `dest/analyzed_commits/` に保存される。
